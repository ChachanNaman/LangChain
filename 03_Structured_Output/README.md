In this i will learn how to process output given by llms

Structured Output : 
->It is well defined data format output(json) rather than free form of text, making models output 
easier to parse and work with code
->textual output generated by output is unstructured 
->we want json enforced output -> structured output 

Why do we need structured Output ?
1. Data Extraction : 
Upload Resume -> text extract -> sends to llm -> llm gets json format(name, place,exp,clg) -> stored in database then 
Amazon Review -> extract keywords from para of reviews -> store pros and cons and sentiments 
2.Agents :
Agents needs tools -> calculator 
agent->prompt->str output(numbers)-> calculator calculates

Ways to get structured output : 
Two types of LLMs-> can generate str output & cant generate str output 
Can -> with_structured_output used to give instruction to llm to give in which way structured output 
Can't -> Output Parsers used 

with_structured_output :
when model invoked after that with_structured_output called with specified data format 
data format -> typedDict, Pydantic, json_schema

typedDict : 
-> is a way to define dictonary in python where you specify what keys and value should exist 
-> ensure that your dictonary follows specific structure
-> first you define class or metadata and with help of this i form dictonaries 
-> doesnt have validation means you can still miswrite the datatype format 

Pydantic : 
-> its a data validation and data parsing library for python. it ensures that data you work with is correct, structured, and type-safe
-> thats why this is used in building fastAPI coz there we need str data
-> there are default values, optional values Optional[int] = None
-> Coerce : Pydantic is smart enough that if we pass str in age then it will automatically parse it to int and print int value 
-> Pydantic have Email vlidation built in(EmailStr) means if i pass wrong formal mail it will throw error
-> Field includes many options for options like cgpa
